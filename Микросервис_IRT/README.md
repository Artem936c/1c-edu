# Микросервис IRT (Item Response Theory)

Stateless-сервис на FastAPI, реализующий математические расчёты моделей IRT
для приложения автоматизированного обучения и тестирования по платформе
«1С: Предприятие» 8.3.

## Назначение

Конфигурация 1С (см. соседнюю папку `../Конфигурация/`) обращается к этому
микросервису по HTTP для решения трёх задач:

1. **Выбор следующего задания** — по критерию максимальной информации Фишера.
2. **Оценка способности θ** — методом максимального правдоподобия (MLE), при
   вырожденном паттерне ответов — MAP-оценкой с априорным N(μ, σ).
3. **Калибровка банка заданий** — оценка параметров b (трудность) и
   a (дискриминативность) методами JMLE / MMLE.

## Структура

```
Микросервис_IRT/
├── pyproject.toml
├── app/
│   ├── __init__.py

│   ├── schemas.py       — Pydantic-схемы запросов/ответов
│   ├── irt_2pl.py       — вероятность и информация Фишера (1PL/2PL)
│   ├── irt_rasch.py     — оценка θ (MLE/MAP, метод Брента)
│   ├── selector.py      — селектор следующего задания
│   ├── calibration.py   — калибровка банка (JMLE/MMLE)
│   └── stopping.py      — критерии остановки сессии
└── tests/
    ├── conftest.py
    ├── test_rasch.py
    ├── test_selector.py
    └── test_calibration.py
```

## HTTP API

Все эндпоинты, кроме `/health`, принимают `POST` с телом JSON.

| Эндпоинт            | Назначение                                   |
|---------------------|----------------------------------------------|
| `POST /estimate_theta`  | оценка θ и SE по истории ответов         |
| `POST /select_next_item`| выбор следующего задания                 |
| `POST /calibrate_bank`  | калибровка параметров банка              |
| `GET\|POST /health`     | проба доступности сервиса                |

### Контракт с конфигурацией 1С

Конфигурация сериализует структуры через `ЗаписатьJSON`, поэтому **ключи
запросов — кириллические** (имена свойств структур 1С). **Ключи ответов —
латинские**, конфигурация читает их по именам `theta`, `se`, `selected_id`,
`new_params`. Это зафиксировано в `app/schemas.py` через `alias`.

`POST /estimate_theta`
```jsonc
// запрос
{ "ИсторияОтветов": [ { "ТрудностьB": 0.4, "ДискриминативностьA": 1.2, "Верно": true } ],
  "Априор": { "Мю": 0, "Сигма": 1 } }
// ответ
{ "theta": 0.83, "se": 0.41, "stop": false, "n_items": 1 }
```

`POST /select_next_item`
```jsonc
// запрос
{ "ТекущаяТета": 0.5,
  "Кандидаты": [ { "Идентификатор": "f1a2…", "ТрудностьB": 0.6, "ДискриминативностьA": 1.4 } ] }
// ответ
{ "selected_id": "f1a2…", "info": 0.49 }
```

`POST /calibrate_bank`
```jsonc
// запрос
{ "Тема": "<id>", "Метод": "JMLE",
  "Матрица": [ { "Пользователь": "Иванов И.И.", "Задание": "f1a2…", "Ответ": 1 } ] }
// ответ
{ "new_params": [ { "item_id": "f1a2…", "b": 0.42, "a": 1.10, "n": 87 } ],
  "method": "JMLE", "n_items": 1, "n_users": 1 }
```

## Запуск локально

```bash
cd Микросервис_IRT
pip install -e ".[dev]"
uvicorn app.main:app --host 0.0.0.0 --port 8001
```

URL микросервиса прописывается в константу `URLМикросервисаIRT` конфигурации 1С
(например, `http://localhost:8001`).

## Запуск в Docker

```bash
cd Конфигурация/deploy
docker compose up -d --build
```

Образ собирается из этого каталога (`context: ../../Микросервис_IRT`),
сервис слушает порт `8001`, healthcheck опрашивает `/health`.

## Тесты

```bash
cd Микросервис_IRT
pip install -e ".[dev]"
pytest
```

Состав проверок:

* `test_rasch.py` — монотонность вероятности, пик информации при θ = b,
  несмещённость и точность оценки θ;
* `test_selector.py` — выбор максимально информативного задания и сходимость
  адаптивной сессии (RMSE ≤ 0.30 логит при длине теста 15 — метрика М-7 ВКР);
* `test_calibration.py` — восстановление параметров банка (Rasch / JMLE / MMLE)
  на синтетических данных 60 заданий × 100+ обучающихся.
