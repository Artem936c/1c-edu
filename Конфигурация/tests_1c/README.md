# Сценарии интеграционного тестирования конфигурации (xUnitFor1C)

Каталог содержит интеграционные сценарии, проверяющие связку общих модулей серверной логики конфигурации на реальной (тестовой) информационной базе «1С: Предприятие» 8.3.

## Состав

- `СценарииИнтеграционногоТестирования.bsl` — модуль-обработка в формате xUnitFor1C с пятью сценариями:
  1. **Прохождение базовой сессии адаптивного тестирования** — `АдаптивноеТестирование.НачатьСессию` → серия `ОбработатьОтвет` → завершение по критерию остановки; проверка статуса сессии и записи итоговой оценки в регистр `ОценкиПодготовленностиПоТемам`.
  2. **Завершение темы и начисление бейджа** — `Геймификация.НачислитьБаллы` → `ПроверитьУсловияБейджей`; проверка появления документа `ВыдачаБейджа`.
  3. **Обновление траектории после прохождения темы** — `ТраекторияОбучения.ПересобратьТраекторию`; проверка записи регистра `СостояниеТраекторииОбучения` и рекомендации следующей темы.
  4. **Пересчёт рейтинга группы** — начисление баллов нескольким обучающимся → `Геймификация.ОбновитьРейтинг`; проверка упорядочивания в регистре `РейтинговыеПозиции`.
  5. **Калибровка банка заданий** — наполнение истории ответов → `РегламентныеЗаданияОбработчики.КалибровкаБанкаЗаданий`; проверка обновления версии параметров в `ПараметрыIRT`. Пропускается, если микросервис IRT недоступен.

## Принципы

- Каждый сценарий **самодостаточен**: создаёт собственные курс, тему, банк заданий с параметрами IRT, при необходимости — бейдж и учебную группу.
- Каждый сценарий выполняется в транзакции, которая **отменяется** после проверок (`ОтменитьТранзакцию`), поэтому тестовая база не засоряется данными прогонов.
- Утверждения используют штатные методы контекста xUnitFor1C: `ПроверкаИстины`, `ПроверкаЛжи`, `ПроверкаРавенства`, `ПроверкаЗаполненности`, `ПометитьПройденным`.

## Запуск

Сценарии не выполняются автономно — им нужна информационная база с загруженной конфигурацией. Порядок:

1. Развернуть тестовую базу и загрузить конфигурацию (см. [../docs/deployment.md](../docs/deployment.md)).
2. Установить фреймворк [xUnitFor1C](https://github.com/xDrivenDevelopment/xUnitFor1C) («Тестер»).
3. Загрузить `СценарииИнтеграционногоТестирования.bsl` как модуль объекта внешней обработки-теста (или подключить через сборку обработки).
4. Запустить набор из «Тестера»; метод `ПолучитьСписокТестов` зарегистрирует все пять сценариев.
5. Для сценария калибровки предварительно поднять микросервис IRT и заполнить константу `URLМикросервисаIRT`, иначе сценарий будет помечен пройденным (пропущен).

> **Примечание о расположении.** В плане реализации (этап 14) каталог обозначен как `configuration/SubsystemTests/`. Фактически сценарии вынесены в отдельный каталог `tests_1c/` вне дерева выгрузки `Conf/`, чтобы не попадать в загрузку конфигурации из файлов как объект метаданных. На состав и назначение сценариев это не влияет.
