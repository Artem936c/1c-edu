# Руководство преподавателя

Руководство описывает типовые сценарии работы преподавателя: формирование учебной группы, наполнение библиотеки и банка заданий, мониторинг прогресса через дашборд, индивидуальный разбор результатов, ручное начисление баллов и выдачу бейджей. Соответствует роли `Преподаватель` (см. [architecture.md, §10](architecture.md)).

## 1. Вход и навигация

1. Запустить «1С: Предприятие», войти под учётной записью с ролью **Преподаватель**.
2. Главное окно — **Рабочий стол преподавателя**: переходы к дашборду, банку заданий, библиотеке и журналу попыток группы.
3. Командный интерфейс разделён на подсистемы: **Курсы**, **Тестирование**, **Библиотека**, **Геймификация**, **Аналитика**, **Администрирование**.

Преподаватель видит и редактирует данные только обучающихся **своих учебных групп** (ограничение RLS).

## 2. Создание учебной группы

1. **Курсы → Учебные группы → Создать**.
2. Заполнить: краткое описание, преподаватель (вы), год набора.
3. В табличной части «Состав» добавить обучающихся.
4. Записать и закрыть. С этого момента вы видите документы и аналитику по участникам группы.

## 3. Формирование курса и тем

1. **Курсы → Курсы → Создать**: краткое и полное описание, преподаватель, даты начала/окончания, целевая аудитория.
2. **Курсы → Темы → Создать**: указать курс-владелец (темы иерархичны и подчинены курсу), описание, сложность, порядок изучения.
3. В табличной части «Пререквизиты» темы перечислить темы, которые должны быть пройдены раньше, — это управляет рекомендациями траектории обучения.

## 4. Наполнение библиотеки

1. **Библиотека → Материалы библиотеки → Создать**.
2. Указать вид материала (учебник, справочник, нормативный документ, методическое пособие, видеоматериал), библиографическое описание по ГОСТ Р 7.0.100–2018, автора, год, издательство, ISBN, ключевые слова.
3. При необходимости присоединить файл материала (подсистема БСП «Работа с файлами»).
4. В табличной части «Привязки к темам» связать материал с темами и задать приоритет — это влияет на рекомендации материалов обучающимся.

## 5. Наполнение банка тестовых заданий

1. **Тестирование → Тестовые задания → Создать**.
2. Указать тему и тип задания: одиночный выбор, множественный выбор, сопоставление, упорядочивание, открытый ответ, фрагмент кода.
3. Заполнить формулировку и контекст предъявления.
4. Для заданий с выбором — табличная часть «Варианты ответа» (текст + признак «Является правильным»); для открытых ответов и фрагментов кода — «Эталонный ответ».
5. Установить признак видимости (`ВидимыйСтатус`) — только видимые задания участвуют в адаптивном подборе.

Форма задания содержит вкладки **«Формулировка»**, **«Параметры IRT»**, **«История ответов»**, **«Калибровка»**.

### 5.1. Параметры IRT и калибровка

- Каждому заданию соответствует запись регистра `ПараметрыIRT`: трудность **b** (от −2 до +2 логит), дискриминативность **a**, статус калибровки и версия параметров.
- Стартовые значения b задаются вручную или загружаются при первоначальном наполнении.
- По мере накопления ответов задание калибруется автоматически: регламентное задание `КалибровкаБанкаЗаданий` пересчитывает b и a (метод JMLE/MMLE) для тем, где на задание накоплено не менее `ПорогОтветовДляКалибровки` ответов, и сохраняет новую версию параметров.
- Статусы: «Не калибровано» → «Идёт калибровка» → «Откалибровано»; «Заблокировано» исключает задание из подбора.

## 6. Практические задания

1. **Тестирование → Практические задания → Создать**: тема, тип проверки, формулировка, эталонный артефакт (присоединённый файл), время на выполнение.
2. Сданные работы фиксируются документом `ВыполнениеПрактическогоЗадания`.
3. Проверка: открыть документ, изучить присоединённое решение, выставить статус проверки, комментарий и итоговый результат.

## 7. Мониторинг прогресса (дашборд преподавателя)

**Аналитика → Дашборд преподавателя** — отчёт с HTML-визуализациями (Chart.js):

- **Распределение θ по темам** — boxplot по каждой теме: видно, какие темы освоены, а какие проблемные.
- **Тепловая карта проблемных тем** — матрица «обучающийся × тема» с цветовым кодированием θ.
- **Активность группы** — стек-диаграмма событий аналитики во времени.
- **Рейтинг группы** — таблица с переходом к карточке обучающегося.

Поддерживается фильтрация по теме, периоду и подгруппе — результат пересчитывается на стороне 1С и перерисовывается в дашборде.

## 8. Индивидуальный разбор результатов

1. Из рейтинга или журнала попыток открыть **документ `ПопыткаТеста`** обучающегося.
2. В табличной части «Ответы» — пошаговая история: задание, ответ, верность, время, текущие θ и SE на каждом шаге.
3. Отчёт **«Отчёт по прохождению теста»** строит кривую сходимости θ, распределение ответов по типам и трудностям, итоговые показатели и рекомендации траектории.

## 9. Ручное начисление баллов и выдача бейджей

### Начисление баллов

1. **Геймификация → Начисление баллов → Создать**.
2. Указать обучающегося, тему, причину `РучноеНачисление`, количество и комментарий.
3. Провести документ — баллы попадают в регистр `БаллыОпыта` и учитываются в рейтинге при следующем пересчёте.

### Выдача бейджа

1. **Геймификация → Выдача бейджа → Создать**.
2. Указать обучающегося и бейдж, дату выдачи, баллы опыта, флаг уведомления.
3. Провести документ.

> Большинство бейджей выдаётся автоматически: регламентное задание `ПроверкаУсловийБейджей` проверяет условия активации из справочника `Бейджи`. Ручная выдача — для нестандартных случаев.

### Настройка правил и наград

- **Геймификация → Правила начисления баллов** — связки «событие → базовые баллы», коэффициент по трудности, признак активности.
- **Геймификация → Бейджи** — категория (стартовый, тематический, мастерский, мотивационный, командный, сезонный), пиктограмма, баллы за получение, табличная часть «Условия активации» (тип условия — параметр — пороговое значение).

## 10. Рейтинг группы

Рейтинг пересчитывается регламентным заданием `ОбновлениеРейтинга` (ежедневно) и хранится в регистре `РейтинговыеПозиции` (сумма баллов, ранг, средняя θ). Для немедленного пересчёта можно запустить задание вручную через «Поддержка и обслуживание → Регламентные операции».
