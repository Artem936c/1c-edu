# Инструкция по развёртыванию

Пошаговое развёртывание системы: подготовка сервера 1С, загрузка конфигурации, запуск микросервиса IRT, настройка параметров, первоначальное наполнение и проверка работоспособности. Общая схема приведена в [README](../README.md); архитектура — в [architecture.md](architecture.md).

## 1. Требования к окружению

| Компонент | Версия / требование |
|-----------|---------------------|
| Платформа «1С: Предприятие» | 8.3.24 и выше |
| Режим совместимости конфигурации | 8.5.1 |
| Вариант встроенного языка | Русский |
| Режим управления блокировкой данных | Управляемый |
| Режим использования модальности | Не использовать |
| Python (микросервис) | 3.11 и выше (образ Docker — 3.12-slim) |
| FastAPI | 0.115 и выше |
| SciPy | 1.13 и выше |
| Docker / Docker Compose | 24.x / v2 |

Сетевое требование: сервер 1С должен иметь сетевой доступ к адресу и порту микросервиса IRT (по умолчанию `http://<хост>:8001`).

## 2. Развёртывание конфигурации 1С

1. Создать пустую информационную базу платформы «1С: Предприятие» 8.3.24 (файловую или клиент-серверную).
2. Открыть базу в режиме **Конфигуратор**.
3. Выполнить **Конфигурация → Загрузить конфигурацию из файлов…** и указать каталог `Конфигурация/`.
4. Дождаться загрузки и выполнить **Конфигурация → Обновить конфигурацию базы данных** (F7). Принять реструктуризацию.
5. (Опционально) Запустить **Конфигурация → Проверка конфигурации** для контроля целостности.

> Конфигурация выгружена в формат XML («Загрузить конфигурацию из файлов»), а не в формат EDT-проекта. Для работы в «1С: EDT» используйте импорт из XML-выгрузки.

## 3. Запуск микросервиса IRT

### Вариант А. Docker Compose (рекомендуется)

Из каталога `Конфигурация/deploy/`:

```bash
docker compose up -d --build
```

Образ `irt-service:0.1.0` собирается из соседнего каталога `../../Микросервис_IRT` (контекст сборки задан в `docker-compose.yml`). Сервис слушает порт **8001**, перезапускается автоматически (`restart: unless-stopped`), доступность контролируется healthcheck по `/health` (интервал 30 с).

Проверка статуса:

```bash
docker compose ps
docker compose logs -f irt_service
```

### Вариант Б. Локальный запуск без Docker

```bash
cd Микросервис_IRT
pip install -e ".[dev]"
uvicorn app.main:app --host 0.0.0.0 --port 8001
```

> На машинах без системного Python используйте `uv` с виртуальным окружением (см. примечание в конце документа).

### Проверка сервиса

```bash
curl http://localhost:8001/health
# {"status":"ok"}
```

Интерактивная документация OpenAPI доступна по адресу `http://localhost:8001/docs`.

## 4. Настройка параметров конфигурации

В режиме **1С: Предприятие** под ролью администратора заполнить константы (раздел «Администрирование» / форма настроек IRT):

| Константа | Синоним | Назначение | Рекомендуемое значение |
|-----------|---------|------------|------------------------|
| `URLМикросервисаIRT` | URL микросервиса IRT | Полный URL сервиса | `http://localhost:8001` |
| `ТаймаутЗапросаКМикросервису` | Таймаут запроса к микросервису, сек | Время ожидания ответа | `10` |
| `ЦелеваяСтандартнаяОшибкаТета` | Целевая стандартная ошибка θ | SE завершения сессии | `0.30` |
| `ЛимитЗаданийВСессии` | Лимит заданий в сессии | Максимум заданий в тесте | `18` |
| `ПорогТетаДляПрохожденияТемы` | Порог θ для прохождения темы | θ успешного прохождения | `0.0` |
| `ПорогОтветовДляКалибровки` | Порог ответов для калибровки | Мин. ответов на задание | `30` |

При незаполненной константе `URLМикросервисаIRT` обращения к сервису не выполняются — система работает в локальном (резервном) режиме расчёта θ.

## 5. Первоначальное наполнение

Эталонные данные лежат в `Конфигурация/fixtures/`:

- `question_bank/*.json` — банк тестовых заданий по шести темам платформы;
- `badges_seed.json` — бейджи с условиями активации;
- `courses_seed.json` — учебный курс с темами и пререквизитами;
- `library_seed.json` — материалы библиотеки.

Запустить обработку `ПервоначальноеНаполнение` (раздел «Администрирование»), указать каталог `fixtures/` и выполнить загрузку. Обработка **идемпотентна**: повторный запуск не создаёт дубликатов. После загрузки в регистре `ПараметрыIRT` появляются стартовые значения параметров заданий.

## 6. Настройка регламентных заданий

В **Конфигураторе** или через «Поддержка и обслуживание → Регламентные и фоновые задания» включить расписания:

| Задание | Рекомендуемое расписание |
|---------|--------------------------|
| `СинхронизацияСМикросервисомIRT` | каждые 5 минут |
| `ПроверкаУсловийБейджей` | каждый час |
| `ОбновлениеРейтинга` | ежедневно, нерабочие часы |
| `КалибровкаБанкаЗаданий` | ежедневно, нерабочие часы |
| `ОчисткаУстаревшихСобытийАналитики` | еженедельно |

Для клиент-серверной базы регламентные задания выполняются автоматически. Для файловой — требуется сеанс с включённым выполнением регламентных заданий.

## 7. Проверка работоспособности

1. **Сервис.** `СинхронизацияСМикросервисомIRT` отрабатывает без ошибок; в журнале регистрации появляется запись о доступности сервиса.
2. **Тестирование.** Запустить адаптивную сессию (рабочий стол обучающегося → «Начать тест») — задания предъявляются, θ и SE пересчитываются на каждом шаге.
3. **Отказоустойчивость.** Остановить микросервис (`docker compose stop irt_service`) и убедиться, что новая сессия продолжается в локальном режиме без необработанных исключений.
4. **Геймификация.** После завершения сессии начисляются баллы, при выполнении условий выдаётся бейдж.
5. **Дашборды.** Отчёты «Дашборд обучающегося» и «Дашборд преподавателя» открываются, диаграммы Chart.js отображаются без ошибок в консоли.

## 8. Обновление

- **Конфигурация.** Повторно «Загрузить конфигурацию из файлов…» из обновлённого `Конфигурация/`, затем «Обновить конфигурацию базы данных».
- **Микросервис.** `docker compose up -d --build` пересобирает образ из актуального кода.

## 9. Диагностика

| Симптом | Вероятная причина | Действие |
|---------|-------------------|----------|
| Тест идёт, но θ «грубая» | Сервис недоступен, работает fallback | Проверить `/health`, значение `URLМикросервисаIRT`, логи контейнера |
| Сбой «Не задан URL микросервиса IRT» | Пустая константа `URLМикросервисаIRT` | Заполнить константу (раздел 4) |
| HTTP-ошибки в журнале регистрации | Неверный URL/порт, недоступность сети | Проверить сетевой доступ сервера 1С к порту 8001 |
| Калибровка не запускается | Мало ответов на задание | Снизить `ПорогОтветовДляКалибровки` или накопить историю |

## Примечание: Python через `uv` (если нет системного Python)

На машинах без системного Python и при заблокированной установке MSI запускайте микросервис в виртуальном окружении, создаваемом `uv`:

```bash
cd Микросервис_IRT
uv venv
uv pip install -e ".[dev]"
uv run uvicorn app.main:app --host 0.0.0.0 --port 8001
uv run pytest
```
